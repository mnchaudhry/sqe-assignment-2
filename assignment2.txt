CLASS 1: Student.java

public class Student {
    private String studentId;
    private String name;
    private double gpa;
    
    public Student(String studentId, String name, double gpa) {
        if (studentId == null || studentId.isEmpty()) {
            throw new IllegalArgumentException("Student ID cannot be empty");
        }
        if (gpa < 0.0 || gpa > 4.0) {
            throw new IllegalArgumentException("GPA must be between 0.0 and 4.0");
        }
        this.studentId = studentId;
        this.name = name;
        this.gpa = gpa;
    }
    
    public String getStudentId() {
        return studentId;
    }
    
    public String getName() {
        return name;
    }
    
    public double getGpa() {
        return gpa;
    }
    
    public boolean updateGpa(double newGpa) {
        if (newGpa < 0.0 || newGpa > 4.0) {
            return false;
        }
        this.gpa = newGpa;
        return true;
    }
    
    public String getGradeLetter() {
        if (gpa >= 3.5) return "A";
        if (gpa >= 3.0) return "B";
        if (gpa >= 2.0) return "C";
        if (gpa >= 1.0) return "D";
        return "F";
    }
    
    public boolean isPassed() {
        return gpa >= 2.0;
    }
}

CONCEPTS EXPLAINED

TESTS 1-10: BASIC JUNIT CONCEPTS
════════════════════════════════════════════════════

assertEquals(expected, actual) - values are equal
• assertNotEquals(unexpected, actual) - values differ
• assertTrue(condition) - condition is true
• assertFalse(condition) - condition is false
• assertNull(object) - object is null
• assertNotNull(object) - object is NOT null
• assertThrows(Exception, code) - code throws exception
@Test - marks a method as a test case
@BeforeEach - runs BEFORE each test (setup)
@AfterEach - runs AFTER each test (cleanup)
@BeforeAll - runs ONCE before all tests
@AfterAll - runs ONCE after all tests

TESTS 11-12: STUB TESTING
════════════════════════════════════════════════════

What is a STUB?
  Simple fake implementation with hard-coded values
  No verification of method calls
  Used to test code that depends on external services


TESTS 13-15: MOCK TESTING
════════════════════════════════════════════════════

What is a MOCK?
  Fake object that tracks method calls
  Verifies HOW methods were called
  Uses Mockito framework


STUB VS MOCK QUICK REFERENCE

┌──────────────────┬─────────────────────┬─────────────────────┐
│ Aspect           │ STUB (Tests 11-12)  │ MOCK (Tests 13-15)  │
├──────────────────┼─────────────────────┼─────────────────────┤
│ Purpose          │ Provide return vals │ Verify behavior     │
│ Implementation   │ Manual stub class   │ Mockito.mock()      │
│ Verify calls     │ NO                  │ YES                 │
│ Complexity       │ Simple              │ More complex        │
│ Framework needed │ NO                  │ YES (Mockito)       │
│ Usage            │ Simple dependencies │ Complex interactions│
└──────────────────┴─────────────────────┴─────────────────────┘

  WHEN TO USE:
  STUB: You only care about return values
  MOCK: You need to verify method calls and interactions
